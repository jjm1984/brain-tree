# Reference: Relationship Conflict Regulation

This is the full backing for `SKILL.md`. The skill file is written to
be read in the moment; this file is written to be read in calm time,
when building or revising the skill itself.

## Why this sequence, in this order

The order matters — it isn't arbitrary. Recognising the shape
(pursuer/withdrawer) comes first because it costs nothing and
prevents the conversation from becoming about who's right.
Regulation comes before borrowing peace because peace can't be
borrowed by a nervous system that hasn't been told what's happening
to it. Inner witness comes *after* regulation, not before, because
self-awareness at the philosophical level requires the prefrontal
cortex to have some access back — which the amygdala-first sequence
(`mycelium-brain`) says isn't guaranteed until regulation has begun.

## Exposed roots

- **Attachment theory (Bowlby / Ainsworth)** — avoidant-style
  deactivation as a candidate account of withdrawal. Untested against
  whether withdrawal here is attachment-specific-to-one-relationship
  or a general stress response. *Speculative, yellow.*
- **Gottman / Christensen — demand-withdraw dynamic** — pursuer and
  withdrawer named directly; source of step 1 and step 5's alignment
  language ("against the problem, not against each other").
- **Polyvagal theory** — ventral vagal tone, safety-first sequencing
  (external safety → nervous system regulation → cognitive
  restoration → philosophical access). Backing for why regulation
  must precede inner witness.
- **RSD (rejection sensitive dysphoria)** — a popular, ADHD-associated
  framing for disproportionate reaction to perceived rejection or
  criticism. *Not a clinical diagnosis.* Held here as one candidate
  explanation for what the nervous system is alarming about — useful
  as a hypothesis to hold loosely, risky if it hardens into a label
  applied to another person from outside. See counter below.

## Counters

- **On RSD as explanation:** naming a pattern is not the same as
  confirming its mechanism. There's a standing caution already in
  place elsewhere in this system — not everything should be
  attributed to a diagnosis, even a real one, and doing so risks
  becoming an excuse-generator rather than a description. Capacity
  being reduced in the moment does not mean character is permanently
  off the table -- see the capacity/character gate sequence
  (session-2026-07-13): capacity is a *precondition* for fair
  character-testing, not an exemption from it, and the two apply in
  sequence, not as a substitute for each other.
- **On co-regulation:** unlike indifferent-order peace, co-regulation
  is not free to the giver. `leaf-withdrawal-repair-pattern` already
  names this directly — the faster-regulating partner "regulates
  themselves, then extends regulation to the other while often still
  hurting themselves, unresolved." A steady one-directional pattern
  risks becoming another instance of the Doing/Coherence engine —
  care metabolised as duty rather than received as reciprocity. Worth
  watching for symmetry over time, not assuming it.
- **On repair generally:** repair is not erasure. The scar stays. This
  applies to the skill as a whole — using it well does not undo
  impact already caused; it changes what happens next.

## Crossroads: co-regulation or space?

*(Full text lives in the branch itself — `leaf-borrowed-peace-co-regulation`
— reproduced here for the skill's own context.)*

Which the moment needs isn't a fixed trait of the person — per the
flight/freeze framing, it's determined by how much self-awareness is
available right then, which is itself the thing regulation is trying
to restore. Open question, deliberately left unresolved: can the
person mid-alarm accurately read their own answer to this fork, or
does asking the question require capacity the alarm has already taken
offline? If the latter, the fork likely needs to be pre-negotiated in
calm time — the same design logic already used for the repair trigger
itself in the seven-step pattern.

## Open threads

- Does co-regulation require rough parity over time, or can one
  partner sustainably carry more of it *if it's acknowledged rather
  than assumed*?
- Is RSD-as-alarm a real, distinguishable mechanism from general
  flight/freeze overload here — or the same thing wearing a more
  specific name? Not yet tested against lived instances.
- Should this skills library live as its own new layer alongside Map,
  or as a Map-layer sub-type (skill files as a specific format of Map
  content, per the Map-Layer Exception Principle)? Structural
  question, not yet resolved — first raised in this session.
- What's the second skill in the library — the kids thread already

  building the next?

## Note on `written-for: human`

This skill is authored assuming a human's embodiment and interiority —
"notice your body," "the alarm goes off before the thinking part of
the brain gets a vote." That's a statement about the artifact's
origin, not a claim about who can act on it. An agent reading this
skill will need to translate steps 2–4 in particular (regulate; borrow
peace; inner witness) into whatever it can actually perceive and do —
what that translation looks like isn't specified here and shouldn't
be guessed at prematurely. Steps 1 and 5 (recognise the shape; return
to the shared problem) are closer to audience-neutral pattern logic
and likely need the least translation.

## Dear reader

This skill was built from the outside in — starting from the
generative machinery of `branch-aliveness` (built originally to
describe solo experiences of peace and flow) and pulled sideways into
relational conflict for the first time. If it reads like it's borrowing
furniture from another room, that's because it is — check
whether the fit is real after it's been lived with, not just argued
for here.
