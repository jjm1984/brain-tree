# Reference: Third-Party Life-Domain Audit Pass

## Provenance

Sourced from `forge/life-audit.md`'s 8-areas framework, staged via
Drive session `ai-brain-session-2026-07-18-skills-layer-resolution-
and-placement.json`. That session correctly reclassified the parent
concept from a Skill candidate to a Map node (`map-life-domain-audit`)
on the reasoning that a cross-domain periodic review is a strategy,
not a single tactic — this file is the tactical execution split out
from that strategy, per this project's Leaf → Fruit+Code → Map →
Skill chain (`meta/principles.md`).

The original 8-area list itself (the specific domains named in the
source Instagram-style framework) was discarded as content — generic,
unattributed, low-tier by this project's own source-verification
standard. Only the one genuinely load-bearing mechanism was kept: the
third-person/external-observer framing. Domains here are mapped onto
this project's own existing branches instead of reproducing an
unattributed external list.

## Why third-person voice is the actual mechanism

This project's own feedback loop has an open, named gap: reading
someone else's motivation is easier than reading your own — a
self-examination-avoidance pattern, confirmed rather than
hypothesized. A first-person life review runs directly into that gap:
the same blind spots that make self-examination hard in general make
a first-person *audit* hard specifically, at the exact moment honesty
matters most.

Third-person framing is a forcing function against that, not a
stylistic choice. Briefing "his" situation to an imagined third party
recruits the same clearer-eyed evaluation this person already uses on
other people's lives — evidenced by the pattern itself being the
reason this skill exists.
