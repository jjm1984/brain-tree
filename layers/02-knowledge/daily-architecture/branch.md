# Daily Architecture

*Peripheral — to be built in the job-change design window.*

Where philosophy meets the alarm clock. The operational layer that translates the code into daily practice: when to think, when to move, when to rest, what comes first, what can't be displaced.

Now sits adjacent to the audit framework — the code provides the criteria for daily architecture. The structure is not built until the code exists to ground it.

## What will live here

- Circadian design: sleep window, ultradian rhythm, light/food/deep work sequencing
- Non-negotiable anchors that survive any surge of urgency
- The physical training window built into structure, not willpower
- Decision-point navigation for the ordinary day

philo-how-to-live (the trunk applied daily), philo-1pct-compounding (the daily anchor), audit-framework (tripwires that require structural conditions to fire), branch-physical-wellbeing (the body systems that daily architecture serves)

## Dear reader

Most people design their days around their demands. This branch holds the question: what does a day look like when it is designed around your values first, and the demands are fitted around that?
